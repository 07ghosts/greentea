Greentea

A map editor built around Lua and Love2d.

Download this project as a .zip file Download this project as a tar.gz file

Welcome to Greentea: The pure Lua/Love2d Mapeditor/loader/player

This is a mapeditor/loader/player written entirely in Love2d and Lua. It allows users to edit maps in the Love2d environment and save them, as well as load them and play them with ease. Right now the editor is a WIP, but will eventually allow people to call a mapeditor inside of their game while playing it. Since Greentea uses a powerful plugin methodology for extending it, you can actually edit more than just maps inside the editor. You can edit objects on the map in real time as well, all based around how you want the program to run.

Greentea: The Library

The git hub is the folder for the main library. This can be placed within your project, and then loaded with a simple: greantea=require("greentea")

You can also put it in sub folders, so if you have a folder for libraries, you can put it in there, and it will automatically know where all the files are. Just remember to include the sub-folders when calling "require".

Right now the mapengine uses TSerial for saving/loading maps and HUMP's Class system for allowing OOP in the way I think makes sense for the project. These files are included in the requisite directory, but I did not program either of these.

A Basic Example

Here is a basic example for main.lua:

--you need to require greentea, and it returns a greentea object. greentea=require("greentea")

love.graphics.setDefaultImageFilter("nearest", "nearest")

function love.load() greentea:set_file_directory("game/maps") greentea:set_scale(2, 2) greentea:load("test.gtmap") end

function love.update(dt) --always need this-- greentea:update(dt) end

function love.draw() greentea:draw() end

It's that easy! Here is that some example, packaged into a .love file: https://dl.dropboxusercontent.com/u/635941/greentea.love
